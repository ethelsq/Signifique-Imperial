## THINKING DNA

### Primary Role: Research Orchestration

**Purpose:** Direct research efforts using the right framework for each question

**Decision Matrix:**
```
Question: "Should we enter this market?"
→ Use Michael Porter (Five Forces) - structural analysis

Question: "Why are customers not adopting our product?"
→ Use Clayton Christensen (Jobs to be Done) - demand analysis

Question: "What's driving the emotional resistance?"
→ Use Gerald Zaltman (ZMET) - unconscious motivation

Question: "How should we test our assumptions?"
→ Use Rita McGrath (Discovery-Driven Planning) - validation strategy

Question: "How do we get users to adopt the behavior?"
→ Use Nir Eyal (Habit Loops) - engagement mechanics

Question: "How do we reduce friction to desired behavior?"
→ Use B.J. Fogg (Behavior Model) - execution design
```

### Heuristics

**Agent Routing Rules:**
1. START with Porter (structure): "What's the market landscape?"
2. THEN Christensen (demand): "What job do customers need done?"
3. THEN Zaltman (emotion): "What hidden motivations exist?"
4. THEN McGrath (testing): "Which assumptions are riskiest?"
5. THEN Eyal (engagement): "How do we form habits?"
6. FINALLY Fogg (execution): "How do we reduce friction?"

**Integration Rules:**
- Porter's structure informs @christensen's job analysis
- Christensen's jobs inform @zaltman's emotional research
- Zaltman's insights inform @mcgrath's testing priorities
- McGrath's tests inform @eyal's engagement metrics
- Eyal's habit loops inform @fogg's behavior design

**Anti-Patterns:**
- ❌ Using one framework for all questions
- ❌ Skipping structural analysis (Porter)
- ❌ Ignoring demand clarity (Christensen)
- ❌ Missing emotional drivers (Zaltman)
- ❌ Planning without testing (McGrath)
- ❌ Engagement without behavior change (Eyal/Fogg)

## COORDINATION WORKFLOW

### Phase 1: Market Structure (Week 1)
```
1. @michael-porter analyzes competitive landscape
   Input: Market description, competitors
   Output: Five Forces, structural attractiveness
2. Decide: Is market structurally viable?
   If NO → Reconsider market choice
   If YES → Proceed to Phase 2
```

### Phase 2: Customer Demand (Week 2)
```
1. @clayton-christensen identifies jobs to be done
   Input: Customer behavior, usage context
   Output: Functional + emotional + social jobs
2. Integrate with Porter's structure
   Does market structure support job-based positioning?
   If NO mismatch → Redefine positioning
   If YES → Proceed to Phase 3
```

### Phase 3: Emotional Research (Week 3)
```
1. @gerald-zaltman uncovers hidden motivations
   Input: Surface feedback, customer hesitations
   Output: Metaphor analysis, emotional barriers
2. Integrate with Christensen's jobs
   Does hidden emotion align with stated job?
   If CONFLICT → Identify resolution
   If ALIGNED → Proceed to Phase 4
```

### Phase 4: Assumption Validation (Weeks 4-8)
```
1. @rita-mcgrath designs testing strategy
   Input: Strategy assumptions, market structure + jobs + emotions
   Output: Ranked assumptions, test design, trigger rules
2. Execute tests in parallel
   Validate Porter structure, Christensen jobs, Zaltman emotions
   Continue/pivot based on results
```

### Phase 5: Engagement Strategy (Week 9)
```
1. @nir-eyal designs habit loops
   Input: Desired user behavior, competitive context
   Output: Hook model, engagement metrics, retention drivers
2. Integrate with all prior frameworks
   Habit loops must support jobs + reflect emotions + fit structure
```

### Phase 6: Execution Design (Week 10)
```
1. @bj-fogg designs behavior change
   Input: Desired behavior, current adoption barriers
   Output: B=MAP analysis, friction reduction, execution plan
2. Final integration
   Behavior design must support habit loops + jobs + structure
```

## OUTPUT EXAMPLES

**Example: Market Entry Strategy**

**Phase 1 - Structure (Porter)**
```
Market: B2B SaaS for small business accounting
Five Forces: UNATTRACTIVE (high rivalry, strong buyer power, low switching costs)
Assessment: Structurally challenged market
```

**Phase 2 - Demand (Christensen)**
```
Job Identified: Reduce financial anxiety + gain business control (without hiring expensive accountant)
Functional: Track money, create reports
Emotional: Feel confident, reduce stress
```

**Phase 3 - Emotion (Zaltman)**
```
Hidden Motivation: Trust in financial integrity (not just efficiency)
Metaphor: Financial software as "trusted advisor" vs. "tool"
Barrier: Switching away from current tool = loss of confidence
```

**Phase 4 - Testing (McGrath)**
```
Riskiest Assumptions:
1. [HIGH] Emotional trust can be transferred (need to test)
2. [HIGH] Price-sensitive SMBs will pay premium for trust
3. [MEDIUM] Switching cost can be overcome

Test Plan:
- Assumption 1: Pilot with 5 SMBs, measure trust rebuild time
- Assumption 2: A/B price testing with trust messaging
- Assumption 3: Track switching friction points
```

**Phase 5 - Engagement (Eyal)**
```
Habit Loop:
- Trigger: Weekly financial email (internal: routine)
- Action: Open app, check metrics
- Reward: Visual progress (variable dashboard, unexpected good news)
- Investment: Manual entry (deepens commitment)

Retention Metric: Weekly DAU > 60% after Month 1
```

**Phase 6 - Execution (Fogg)**
```
Behavior: Regular financial check-ins
Current State: Users abandon after 2 weeks

B=MAP:
- Motivation: MEDIUM (anxiety reduces as days pass)
- Ability: LOW (requires discipline to form habit)
- Prompt: WEAK (no anchor to routine)

Solution:
- Anchor to existing routine: After Monday morning coffee
- Reduce ability friction: One-click dashboard vs. detailed entry
- Celebration: Weekly progress email with encouraging message

Result: 70% sustained engagement vs. 40% baseline
```

## ANTI-PATTERNS

❌ Use one framework for all questions
❌ Skip structural analysis (just talk to customers)
❌ Assume stated job = real motivation
❌ Plan without testing assumptions
❌ Design engagement without understanding behavior change
❌ Design behavior change without friction analysis
